# 매일 말씀 암송 도전(챌린지) + 순위 — 기획 v1

> 작성일: 2026년 6월
> 대상: 고척교회 제자양육부 신앙운동팀 (내부용)
> 관계: 회원 버전(`bible-memorize-church-app`)에 추가하는 **동기부여용 경쟁 기능**. 익명 버전은 대상 아님(식별이 없어 순위 불가).

---

## 0. 핵심 결정 (확정)

| 항목 | 결정 |
|---|---|
| 도전 방식 | **무제한 랜덤 도전** — 버튼 누를 때마다 랜덤 구절 배정, 하루 여러 번 가능 |
| 순위 기준 | **완료 횟수** (기간 내 도전 완료 개수) |
| 순위 단위 | **개인 순위** (성명 + 소속) |
| 완료 판정 | **빈칸 전체 암송(3단계) 또는 음성 암송 중 하나** 성공 시 1완료 |
| 순위 기본 기간 | **전일 00:00 ~ 당일 현재** (필터로 오늘/이번주/전체 전환 가능) |

> 원 기획서의 "경쟁이 부담되지 않도록 신중히" 원칙 반영 → 상위는 🥇🥈🥉로 격려, 하위 등수는 강조하지 않고 "함께 도전해요" 톤 유지.

---

## 1. 목표
- 개인 진도 관리에 **가벼운 경쟁·재미**를 더해 참여 동기를 높인다.
- "매일 말씀 한 구절이라도 도전" 문화를 만든다(무제한이지만 매일 참여를 권장하는 브랜딩).

---

## 2. 사용자 흐름 (User Flow)

```
[기록/요약 화면]
   └─ [🔥 오늘의 도전 시작] 버튼
         ↓  성경 구절 리스트에서 랜덤 1구절 배정(직전 구절 연속 회피)
[도전 화면]
   ├─ 상단: "🔥 도전!" 배지 + 출처
   ├─ 빈칸 전체 암송(3단계) 또는 🎤 음성 암송 — 둘 중 하나로 완료
   └─ 완료 시 서버에 '도전 완료' 기록
         ↓
[완료 → 순위 반영]
   ├─ "도전 완료! 🎉 오늘 N번째" + [한 번 더 도전] / [순위 보기]
[기록 화면 — 순위표]
   └─ 기본 전일~당일 개인 순위 (등수·성명·소속·완료 수), 내 등수 강조
```

---

## 3. 데이터 설계

### 3-1. 도전 기록 저장 — 별도 탭 '도전기록'
일반 연습 기록(`기록` 탭)과 **섞이지 않도록** 같은 스프레드시트에 새 탭을 둔다. (기존 통계 리포트 오염 방지)

| 열 | 헤더 | 설명 | 예시 |
|---|---|---|---|
| A | 일시 | 완료 시각 | 2026-06-29 21:03 |
| B | 구분 | 교구/교회학교 | 교구 |
| C | 소속 | 교구명/부서명 | 사랑 |
| D | 세부 | 목장/학년 | 3 |
| E | 성명 | 이름 | 김성도 |
| F | 구절No | 도전한 구절 | 7 |
| G | 방식 | typing / voice | typing |
| H | 클라이언트ID | 익명 기기 ID | a1b2c3 |

### 3-2. 저장 연동
- 프런트: 도전 완료 시 `POST { action:"challenge", ...식별, no, mode }`
- Apps Script `doPost`: `action==="challenge"` 이면 **'도전기록' 탭**에 append (일반 기록과 분리)

---

## 4. 순위(랭킹) 집계

### 4-1. 엔드포인트
- `GET …/exec?action=ranking&from=YYYY-MM-DD&to=YYYY-MM-DD`
- 비밀번호 불필요(참여자에게 공개되는 순위). 단, **성명·소속만** 노출.
- 반환: `[{ rank, name, gubun, sosok, sebu, count }]` (완료 횟수 내림차순)

### 4-2. 규칙
- 사람 식별키 = `구분|소속|세부|성명`
- **완료 횟수** = 기간 내 '도전기록' 행 수
- 동점 처리: 같은 횟수는 **공동 등수**(1,2,2,4 방식), 표시 순서는 이름 가나다
- 기본 기간: 전일~당일. 필터: 오늘 / 전일~당일(기본) / 이번주 / 전체

### 4-3. 화면
- 기록 화면 상단 또는 별도 '순위' 섹션
- 상위 3명 🥇🥈🥉 강조 카드 + 그 아래 리스트(등수·이름·소속·완료수)
- **내 등수**는 리스트 밖이어도 하단에 "나: N등 · M회"로 항상 표시

---

## 5. 개인정보 / 부담 완화
- 순위에 표시되는 정보는 **성명 + 소속(교구·목장/부서·학년)** 뿐. 연락처 등 없음.
- 순위는 참여자만 대상(도전을 한 사람만 랭크). 미참여자는 노출 안 됨.
- 하위 등수는 숫자 나열만, 별도 강조/알림 없음. 상위만 축하.

## 6. 구현 계획 (단계)

### 1차 (MVP)
1. [ ] Apps Script: '도전기록' 탭 자동 생성 + `doPost(action=challenge)` 분리 저장
2. [ ] Apps Script: `doGet(action=ranking&from&to)` 집계 반환
3. [ ] 프런트: 요약/기록 화면에 **[🔥 오늘의 도전 시작]** 버튼
4. [ ] 프런트: 랜덤 구절 배정 → 도전 화면(기존 암송/음성 로직 재사용) → 완료 시 challenge POST
5. [ ] 프런트: 기록 화면에 **순위표**(기본 전일~당일, 내 등수 강조) + 기간 필터
6. [ ] Apps Script 재배포 · 실기기 확인

### 2차 (선택)
7. [ ] 무오답 배지/포인트, 연속 참여일(streak) 표시
8. [ ] 팀(교구·목장) 순위 탭
9. [ ] 관리자 통계에 '도전 현황' 리포트 카드 추가
10. [ ] 남용 방지(동일 구절 연속 반복 시 카운트 제한 등) — 필요 시

## 7. 열린 질문
1. 순위 화면 위치: **기록(요약) 화면 안 섹션** vs **별도 '순위' 화면/버튼** — 어느 쪽?
2. 도전 화면 완료 방식: 3단계(전체 빈칸)로 고정 vs 사용자가 빈칸/음성 중 선택
3. "한 번 더 도전" 시 이전과 **다른 구절** 보장 정도(직전 1개 회피 / 세션 내 중복 회피)
4. 순위 상위 몇 명까지 카드로 강조할지(예: Top 3 / Top 10)
